# Food Client 

Frontend-часть современного веб-приложения для поиска и просмотра рецептов, построенная на **Next.js** (App Router). Проект демонстрирует использование серверного рендеринга (SSR), инкрементальной статической регенерации (ISR) и продвинутого управления состоянием через MobX.

[**Посмотреть Live Demo**](https://next-food-client-pi.vercel.app/)

---

## Технологический стек

* **Framework:** [Next.js](https://nextjs.org/) (App Router, Server & Client Components)
* **Language:** [TypeScript](https://www.typescriptlang.org/) (Строгая типизация моделей данных)
* **State Management:** [MobX](https://mobx.js.org/) (Паттерн Local Stores с кастомным хуком `useLocalStore`)
* **Styling:** SCSS Modules + [classNames](https://www.npmjs.com/package/classNames)
* **Data Fetching:** Fetch API с интеграцией ISR (Revalidation)
* **Backend:** Интеграция с Strapi CMS (использование сложных запросов через `populate` и фильтрацию)
* **Утилиты:** `qs` для сериализации параметров запросов в URL

---

## Основные возможности

* **Личный кабинет:** Регистрация и вход в аккаунт через JWT токен.
* **Избранное:** Избранные рецепты храняться отдельным списком.
* **Гибридный рендеринг:** Сочетание Server Components для быстрой загрузки и SEO с интерактивными Client Components там, где это необходимо.
* **Умная фильтрация:** Выбор категорий синхронизирован с URL Search Params. Это позволяет сохранять состояние фильтров при перезагрузке страницы или отправке ссылки другу.
* **Динамический расчет порций:** Позволяет расчитать необходимое количество ингредиентов на указанное количество порций. 
* **Список покупок:** Инструмент для автоматического формирования списка покупок, согласно динамическому расчету порций, и управления им. 
* **Интерактивный шеф-повар:** Режим интерактивной готовки блюда по шагам с таймером.
* **Кулинарный календарь:** Календарь, позволяющий запланировать приготовление рецептов с помощью drag-and-drop по дням недели.
* **Темная тема:** Сервис поддерживает светлую и темную тему.
* **Адаптивная верстка:** Адаптивная верстка под все виды устройств.
* **Производительность и UX:**
    * Использование `<Suspense>` и `Loader` для обработки состояний загрузки.
    * Оптимизированные запросы к API с кэшированием на стороне сервера.
    * Плавный скролл и адаптивные интерфейсы.

## Структура проекта

Проект следует принципам модульности, где основная бизнес-логика и переиспользуемые элементы вынесены в `shared`:

```
next-food-client
├─ src
│  ├─ app
│  │  ├─ (auth) # Авторизация
│  │  │  ├─ login # Страница логина
│  │  │  └─ register # Страница регистрации
│  │  ├─ (protected) # Защищенные страницы (только для авторизованного пользователя)
│  │  │  ├─ favorites # Страница избранное
│  │  │  ├─ meal-planner # Страница кулинарный календарь
│  │  │  └─ profile # Страница профиль пользователя
│  │  ├─ error.tsx # Компонент ошибки
│  │  ├─ not-found.tsx # Компонент не найденой страницы
│  │  ├─ page.tsx # Страница списка рецептов
│  │  ├─ recipe 
│  │  │  └─ [id] # Страница рецепта
│  │  └─ shopping-list # Страница списка покупок
│  └─ shared
│     ├─ components # UI-компоненты
│     ├─ config # Конфигурация маршрутов (routes) и глобальных настроек
│     ├─ hooks # Кастомные хуки (useLocalStore)
│     ├─ lib # Запросы для серверных компонентов
│     ├─ store # MobX сторы и модели данных с логикой нормализации
│     ├─ styles # Глобальные стили
│     ├─ types # Общие интерфейсы и типы данных TypeScript
│     └─ utils # Утилитарные функции
├─ tsconfig.json
└─ yarn.lock

```

## Быстрый старт

### 1\. Клонирование репозитория

```bash
git clone [https://github.com/D1n8/next-food-client.git](https://github.com/D1n8/next-food-client.git)
cd next-food-client
```

### 2\. Установка зависимостей

*Важно: из-за специфики зависимостей в некоторых окружениях рекомендуется использовать флаг `--legacy-peer-deps` для избежания конфликтов версий ESLint.*

```bash
npm install --legacy-peer-deps
```

### 3\. Настройка окружения

Создайте файл `.env.local` в корне проекта и укажите адрес вашего API (Strapi):

```env
NEXT_PUBLIC_BASE_URL=[https://your-strapi-instance.com](https://your-strapi-instance.com)
```

### 4\. Запуск в режиме разработки

```bash
npm run dev
```

Откройте [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000) в браузере.
